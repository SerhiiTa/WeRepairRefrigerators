# WeRepairRefrigerators Coding Agent Instructions

## Project Goal
Build a secure AI-powered refrigerator repair technician platform.

Initial MVP:
- Houston market
- Refrigerator repair only
- Technician profiles
- Repair case creation
- AI-generated SEO repair articles
- Secure dashboard

## Tech Stack
- Next.js App Router
- TypeScript
- Tailwind CSS
- ESLint
- Supabase later for auth/database

## Coding Rules
- Use TypeScript.
- Use clean component structure.
- Keep pages simple and production-ready.
- Do not add random libraries without approval.
- Do not hardcode API keys.
- Do not create .env files with secrets.
- Never commit node_modules.
- Use semantic HTML.
- Use mobile-first responsive design.
- Keep security in mind.

## Current Priority
Build the frontend MVP:
1. Homepage
2. Technician dashboard placeholder
3. New repair case form
4. Repair case preview page
5. Basic layout/navigation

## Design Style
Professional, clean, dark SaaS style.
Audience: appliance repair technicians and service business owners.

## Important
Do not implement payments, marketplace, live chat, dispatch, or complex AI yet.
Focus only on the Houston refrigerator repair MVP.
